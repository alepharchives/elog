Overview

elog is very basic logging library for Erlang. Its main advantage is its low foot-print. It was tested on a very log-intensive application running on a server with 128K concurrent clients and it worked smoothly.

Usage

  1. First you should include "elog.hrl" in each module you wish to log from
  2. To log something, just use ?LEVEL(Str, Args) macros on your code. The available levels are:
    • DEBUG
    • INFO
    • STAT
    • WARN
    • ERROR
    • THROW (Which behaves like ERROR but throws and catches an ad-hoc exception to let you see the real stack trace when you're out of an exception catch block)
    • FATAL
  3. You can use no_LEVEL macros to avoid the log calls of level LEVEL when compiling
  4. You can use no_log to compile your code with no log at all
  5. To start logging, start the app (i.e. elog:start(). or application:start(elog).)
  6. To decide what to log by default you need to configure the App
  7. You have several elogger options to choose in order to decide how to log:
    • Use elogger_console to log into the console
    • Use elogger_file if you want your logs into archives
    • Use elogger_mail to spam peopl... I mean... to send a mail for each log message
    • You can also create your own elogger implementation to suit your needs
  8. We've added a couple of convenient functions to elog to let our users log better. Check them out
  9. If you want to trap error_logger messages or even replace it altogether, you should check elog_handler

Configuration

elog uses Erlang config files to set its environmental variables. The allowed options are:
level
Default log level: nothing below that level will be logged unless an specific logger is started for that
logger
Default logger: unless otherwise specified (see below), this elogger implementation will be used to handle all logging
LEVEL
Specific configuration for LEVEL. Each log level may have its own configuration. The allowed options are:
logger
Logger: {link @elogger} implementation to be used for this level

Performance Notice

Note that when you decide to set the log level for a module, every other module compiled with that level on will start sending messages to the logger. The logger will ignore them anyway, but if you're concerned about the amount of messages sent through the system you should use this feature with extra care
